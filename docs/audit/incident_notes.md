# Incident Notes

Use this file to capture operator decisions before final push or merge.

## Objective
- What decision are you trying to make?

## Non-Negotiables
- Constraints that cannot be violated.

## Definition Of Good
- What must be true before pushing?

## Commits To Preserve
- Commits that should remain unchanged.

## Commits Or Changes To Question
- Any commit or file group needing extra review.

## Known Risks
- Risks already accepted or under investigation.

## Open Questions
- Questions requiring manual verification.

## Final Decision
- Keep, split, drop, or defer.
